182p 컬렉션 프레임워크

배열의 안 좋은 점
-중간에 하나가 빠지면 다 앞으로 땡겨야함
-맨앞에 하나 들어오면 나머지를 다 뒤로 땡겨야함
-추가 삭제가 어렵다

그래서 자바에서는 컬렉션이라는게 있음
제공되는 거라서 프레임워크

제일위에는 object

collection
-값만 저장
-인터페이스
map
-키는 중복되면 안됨(키로 구분을 함)
-키랑 값 저장
-인터페이스
list
-값만 저장
-인덱스로 구분
-인터페이스 
-중복 값허용
-추가된 순서가 유지
 arraylist
  -내부적으로 순서대로 저장
 linkedlist
  -기차랑 똑같음
  -동적으로 바뀜
set
-값만 저장
-인터페이스
-입력한 순서 기억 못함(순서 개념이 없음)
-중복 허용 하지 않음

제너릭
제너릭은 컬렉션에서 많이 씀
